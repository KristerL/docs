The Docs Service simplifies and standardizes component documentation. It loads doc files, from each Component Directory, and generates each component documentation according to the template set by the its Env.

The docs are presented in the 'Overview' tab of the Workspace UI/Scope UI.

## Loading the component docs

The docs service automatically loads component docs by searching in each Component Directory for files named with certain patterns.
The default glob pattern for doc file is: `**/*.docs.*`.

To add other glob patterns use the Docs config interface:

```json title="workspace.jsonc"
{
  "teambit.docs/docs": {
    "pattern": ["**/*.docs.*"]
  }
}
```

## The docs structure

### Display name

The display name is determined by the [component name](/components/component-id#component-name).

### Abstract

The abstract is a one or two liner description of a component. It should characterize the component's main concern.

By default, the abstract is parsed out from the component's JSDocs annotations. It can be manually set using the `abstract` API:

```js title="heading.docs.js"
export const abstract = 'Renders a heading with customizable element prop';
```

### Labels

Keywords that describe and categorize the component. Labels are also used by [Bit Cloud](https://bit.dev)'s search and filters.

To manually set the labels use the `labels` API:

```js title="heading.docs.js"
export const labels = ['H1', 'H2', 'H3', 'H4', 'H5', 'H6'];
```

### Content

The body of the component documentation. This is where the unstructured content goes.

Set the content by exporting, as default, a component. The type of component varies according to the Env being used. For example, when using the React Env, a React component will be exported.

```js title=heading.docs.js
import React from 'react';

export default function Content() {
  return (
    <>
        <h3>Heading<h3>
        <div>
            A heading component that renders a different heading element depending on
            what you pass into it.
        </div>
    <>
  );
}
```

### Live examples

Live examples are small interactive tutorials that instruct on how a component can be used.

Use the `example` API to set an array of examples.

```ts
Example: {
    scope: object, // All dependencies necessary for the live example to run
    title: string, // The example's title
    description: string, // The example's description
    code: string // A template literal for the example code
}

examples: Example[]
```

For example:

```jsx title="card.docs.tsx"
import React from 'react';
import { Header } from './header';

export const examples = [
  {
    scope: {
      Header,
    },
    title: 'Live Playground',
    description: 'Modify the text to see it change live:',
    code: `<Heading element="h2">
                heading<div>more heading</div>
            </Heading>`,
  },
];
```

## Doc Templates

Doc templates are apps that present doc files content using a specific layout, and may also enrich it with additional data provided by the templates.
Templates are set by Envs to provide doc automation that best fits the Envs components.

For example, the React Env uses the [`@teambit/react.ui.docs-app`](https://bit.dev/teambit/react/ui/docs-app) template to display react docs.
This template is able to render React components, and provides layout and theme for MDX files. Moreover, the template enriches the docs with a 'property table' that presents the components prop types.

See here, to learn how to create your own doc templates.
